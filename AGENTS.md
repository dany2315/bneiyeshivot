# Instructions projet

- Tout texte visible par l’utilisateur ajouté ou modifié dans le site, l’admin, les emails, les PDF, les exports, les toasts, les placeholders, les métadonnées et les messages d’erreur doit être relu avant validation.
- Utiliser une orthographe française correcte, les accents, la ponctuation française et les apostrophes typographiques (`’`) dans les chaînes visibles.
- Ne pas modifier les identifiants de code, variables, fonctions, imports, URLs, slugs, clés JSON, valeurs enum, classes Tailwind ou structure JSX sauf demande fonctionnelle explicite.
